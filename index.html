Issues & Recommendations

Each annotated pin expanded into a full issue brief with context, impact, and a concrete fix.

01
No Responsive Navigation for Mobile
UXAccessibility

The navigation bar collapses poorly on screens below 768px. Items are clipped or invisible. Over 60% of non-profit site traffic now comes from mobile devices — this directly reduces donation conversions.

Recommended Fix
Implement a hamburger menu with a slide-out drawer. Ensure touch targets are at least 44×44px. Test on iOS Safari and Android Chrome.
Priority
High
02
Donate Button Has Poor Visual Hierarchy
UXVisual

The primary "Donate" call-to-action blends in with the surrounding navigation links. It doesn't stand out sufficiently, causing users to miss it — especially in a crowded navigation bar.

Recommended Fix
Style the Donate button with a high-contrast background (e.g. amber #F5A623 on dark navy) and increase padding. Add a subtle animation on page load to draw attention.
Priority
High
03
Hero Section Lacks Trust Signals
UXCredibility

First-time visitors see a generic hero banner with no impact numbers, testimonials, or certifications. Trust is the biggest conversion lever for non-profits — without it, bounce rate stays high.

Recommended Fix
Add impact stats (e.g. "₹2.4Cr raised · 8,000 lives changed") below the headline. Include NGO certification badges and a brief founder quote. Test with 2–3 variations.
Priority
High
04
Single CTA With No Secondary Path
UXConversion

The hero presents one action — "Donate Now" — with no softer secondary option. Visitors who aren't ready to donate have nowhere to go, increasing exit rates from people who could become future donors.

Recommended Fix
Add a ghost/outline secondary CTA: "Learn Our Story →". This gives undecided visitors a path to explore without feeling pressured, improving session depth.
Priority
Medium
06
All Program Images Missing Alt Text
AccessibilitySEO

Every image in the program cards section has an empty or absent alt attribute. This causes WCAG 2.1 Level A failures and removes meaningful content for search engine indexing.

Recommended Fix
Audit all <img> tags and write descriptive alt text (e.g. "Students attending literacy workshop in rural Maharashtra"). Aim for under 125 characters per description.
Priority
High
07
Text Contrast Fails WCAG AA
AccessibilityVisual

Secondary body copy is rendered in light grey (#a0a0a0) on a white background — achieving a contrast ratio of only 2.8:1, far below the required 4.5:1 for normal text.

Recommended Fix
Darken secondary text to at least #757575 (4.6:1 ratio). Use the WebAIM Contrast Checker during design reviews. Set a CSS variable for muted text to apply the fix globally.
Priority
High
08
Footer Lacks Social Media Links
UXSEO

The footer contains copyright and navigation links but no social media icons or handles. Social presence is critical for non-profits to build community, share impact stories, and attract new donors.

Recommended Fix
Add iconographic social links (Instagram, Facebook, Twitter/X, YouTube) with appropriate aria-labels. Link to active accounts and consider adding a "Follow us for impact updates" prompt.
Priority
Medium
09
No Email Newsletter Capture
UXConversion

There's no mechanism to capture visitor emails. Email remains the highest-ROI channel for non-profit fundraising. Without a list, the foundation cannot run campaigns, impact updates, or year-end fundraising drives.

Recommended Fix
Add a simple email signup in the footer or as a mid-page interstitial: "Get monthly impact reports →". Integrate with Mailchimp or equivalent. Offer a free resource download as incentive.
Priority
Medium
05
Program Card Layout Is Clean ✓
Strength

The three-column grid layout for program showcases is well-structured, creating clear visual separation between initiatives. The card pattern scales naturally for adding more programs.

Opportunity to Enhance
Preserve this layout but add a hover interaction (gentle lift + color accent) and a "Learn More →" link on each card. Consider a filter/tab system if programs grow beyond 6.
Priority
Low

3-Phase Improvement Plan

A phased rollout approach to avoid disruption while steadily improving the site's effectiveness.

⚡ Phase 1 — Immediate (Week 1–2)
Fix all image alt attributes across site
Update text contrast CSS variables to WCAG AA
Restyle Donate CTA button (color + size)
Add social media icons to footer
Fix broken links & 404 errors
🔧 Phase 2 — Core Fixes (Week 3–6)
Implement responsive mobile nav with hamburger
Redesign hero with impact stats + dual CTA
Add registration & certification badges
Improve page load speed (image compression)
Add keyboard focus states throughout
🚀 Phase 3 — Growth (Month 2–3)
Build & integrate email newsletter capture
Add program card hover interactions
Implement analytics & conversion tracking
A/B test hero section variants
Conduct full accessibility WCAG 2.1 AA audit
